# Technologies utilisées pour le projet

|Couche|Technologie|Justification|
|---|---|---|
|Front-end|React |J’utilise React afin de créer une interface utilisateur dynamique, réutilisable et performante grâce à son architecture basée sur les composants.|
|Routing front| React Router| Standard de facto pour le routing client en React|
|Styling|SCSS|J’utilise SCSS afin de structurer et maintenir plus facilement mes styles grâce aux variables, aux mixins et à l’imbrication du code CSS.|
|Back-end|Node.js + Express|J’utilise Node.js afin de développer des applications backend rapides et scalables en JavaScript, avec une gestion efficace des opérations asynchrones et des requêtes serveur.|
|Base de données|PostgreSQL|Robuste, relationnel, idéal pour les données structurées avec relations|
|ORM|Prisma|J’utilise Prisma plutôt que Sequelize afin de profiter d’un typage natif, d’une meilleure expérience développeur et d’une gestion des requêtes et des migrations plus moderne et intuitive|
|Authentification|JWT + bcrypt|Standard pour les API stateless, hashage sécurisé des mots de passe|
|API externe |???|Données riches sur les films et séries, gratuite, large catalogue|
|Conteneurisation|Docker + Docker Compose|Environnement de développement reproductible|
|Versionning|Git + GitHub|Standard industriel, workflow collaboratif|
|Déploiement|Render / Railway (back), Vercel / Netlify (front)|Hébergement gratuit adapté aux projets pédagogiques, CI/CD intégrée|
|Tests |Jest (unitaire), Supertest (API), Playwright (E2E)|Couverture complète, écosystème JavaScript|
